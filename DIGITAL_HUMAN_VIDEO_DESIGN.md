## 一、方案 A：2D 数字人融入现有视频（近期可落地）

### 1.1 目标

用户上传一段已有视频 + 选择已创建的 2D 数字人形象 → 数字人被合成到视频中，口型与视频原声同步，作为画中画或前景角色。

### 1.2 架构

```
输入: 已有视频(MP4) + 数字人形象(avatar_id) + TTS文本(可选)
                                 ↓
    ┌─────────────────────────────────────────────┐
    │         2D 数字人嵌入引擎                     │
    │                                              │
    │  1. 视频解析 → 提取帧序列 + 音频轨道          │
    │  2. 如果提供了TTS文本 → TTS合成 → 替换音频    │
    │  3. 如果使用原声 → 提取音频 → 口型分析        │
    │  4. AvatarAnimateStage → 生成数字人口型视频   │
    │  5. 合成引擎：把数字人叠加到原视频             │
    │     - 支持位置：右下角画中画 / 左侧前景        │
    │     - 支持绿幕/透明背景                       │
    │     - 支持大小缩放                           │
    │  6. 输出最终视频                             │
    └─────────────────────────────────────────────┘
                                 ↓
                    输出: 合成后视频(MP4)
```

### 1.3 前后端设计

#### 后端 API

| 端点 | 方法 | 功能 |
|:----|:----:|------|
| `/api/dream-gallery/embed` | POST | 提交视频+数字人+配置 |
| `/api/dream-gallery/embed?id=` | GET | 轮询结果 |

请求参数：
```
file:          原始视频 (MP4)
avatar_id:     数字人形象 ID
voice_id:      声音 ID (可选，不传则用原声)
text:          TTS 文本 (可选，不传则保留原声)
position:      "bottom-right" | "bottom-left" | "overlay" (位置)
scale:         0.2-0.5 (数字人在画面中的占比)
keep_audio:    true/false (是否保留原音频)
```

### 1.4 后端 Python 脚本核心逻辑

```python
def embed_avatar(video_path, ref_path, avatar_meta, audio_path, position, scale):
    # 1. 用 AvatarAnimateStage 生成数字人口型视频
    animator = AvatarAnimateStage()
    ctx = PipelineContext(...)
    ctx.set_artifact(1, "audio", audio_path)
    animator.process(ctx, ModelManager())
    avatar_clip = ctx.get_artifact(1, "clip")
    
    # 2. 用 ffmpeg 把数字人叠加到原视频
    # 计算位置: 右下角
    cmd = [
        "ffmpeg", "-i", video_path, "-i", avatar_clip,
        "-filter_complex",
        "[1:v]scale=iw*SCALE:ih*SCALE,colorkey=0x00ff00:0.3:0.1[fg];"
        "[0:v][fg]overlay=W-w-20:H-h-20[out]",
        "-map", "[out]", "-map", "0:a?",
        output_path
    ]
```

## 二、方案 B：3D 真人形象剧（中长期）

### 2.1 目标

AI 生成 30 分钟的 3D 真实人物形象剧，用户创建的 3D 数字人可以作为角色加入剧中。

### 2.2 核心挑战

| 挑战 | 说明 |
|:-----|:------|
| **3D 形象创建** | 从照片生成可动画的 3D 人脸模型 |
| **3D 场景** | 生成一致的 3D 场景（室内/室外/古装等） |
| **动作动画** | 角色行走、手势、表情变化 |
| **多角色一致性** | 同一角色在不同场景外观一致 |
| **渲染速度** | 30 分钟视频 × 30fps = 54000 帧 |
| **macOS 本地运行** | 全部在 Apple Silicon 上运行，不依赖云 |

### 2.3 建议架构

```
┌─────────────────────────────────────────────────┐
│              3D 数字人剧 Pipeline                 │
├─────────────────────────────────────────────────┤
│                                                   │
│  阶段 1: 剧本生成 (已有能力)                       │
│  ┌───────────────────────────────────────┐       │
│  │ StoryIngest → PromptExpand → 场景JSON │       │
│  └───────────────────────────────────────┘       │
│                                                   │
│  阶段 2: 3D 角色创建                              │
│  ┌───────────────────────────────────────┐       │
│  │ 照片 → 3D人脸重建 (MLX/NeRF) → FBX    │       │
│  │ 或简单方案: 2D数字人 + 伪3D效果        │       │
│  └───────────────────────────────────────┘       │
│                                                   │
│  阶段 3: 场景生成                                  │
│  ┌───────────────────────────────────────┐       │
│  │ 描述 → 场景背景图 (Flux) → 3D场景渲染  │       │
│  └───────────────────────────────────────┘       │
│                                                   │
│  阶段 4: 动作编排                                  │
│  ┌───────────────────────────────────────┐       │
│  │ 脚本 → 动作序列 → 口型驱动 → 动画渲染  │       │
```

### 2.4 推荐路线：2.5D 伪 3D 方案（最务实）

**不建议直接从零开始做真 3D**，在 Apple Silicon 上用 MLX 做 3D 渲染工程量极大。推荐走 **2.5D 伪 3D 方案**：

```
┌────────────────────────────────────────────────────┐
│              2.5D 伪 3D 方案                        │
├────────────────────────────────────────────────────┤
│                                                     │
│  1. 形象: 2D 数字人 + 多层深度分离                    │
│     - AvatarAnimateStage 生成口型同步视频            │
│     - 把人物从背景抠出 (已有抠图工具)                │
│     - 叠加到 3D 风格场景图上                        │
│                                                     │
│  2. 场景: Flux 生成场景背景图                        │
│     - 每个场景生成一张宽幅背景 (1024×576)            │
│     - Ken Burns 效果模拟镜头运动                     │
│     - 可选: 简单前景层(树木/家具) 增加景深感         │
│                                                     │
│  3. 动作: 预置动作模板                              │
│     - 说话: 口型动画 (已有)                         │
│     - 行走: 横向位移 + 手臂摆动 (简单动画)           │
│     - 转身: 左右翻转 (简单)                          │
│                                                     │
│  4. 多角色: 每个角色独立数字人形象                     │
│     - 对话场景: 分屏/左右布局                        │
│     - 每人一个 AvatarAnimate 实例                   │
│                                                     │
│  5. 渲染: ffmpeg 合成                               │
│     - 逐场景渲染 → 拼接 → 最终30分钟视频            │
│     - 预估渲染时间: 30分钟视频 ≈ 2-3小时            │
│                                                     │
└────────────────────────────────────────────────────┘
```

### 2.7 技术选型建议

| 需求 | 建议方案 | 理由 |
|:-----|:---------|:-----|
| 3D 人脸重建 | **MLX + 3DMM** | 纯 Apple Silicon 运行 |
| 场景背景 | **Flux 1.0** | 已有，1024×576 即可 |
| 动作驱动 | **MediaPipe Pose** | 轻量级姿态估计 |
| 渲染引擎 | **ffmpeg + OpenCV** | 已有，零额外依赖 |
| 长视频 | **分场景渲染 + concat** | 已有 pipeline 支持 |

## 三、决策建议

### 短期（立即开始）

**方案 A：2D 数字人嵌入现有视频**

理由：
- 全部基于已有能力（AvatarAnimate + ffmpeg）
- 零额外模型依赖
- 用户可见的实用功能
- 4天可交付

### 中期（方案A之后）

**从 2D 过渡到 2.5D 伪 3D 剧**

路线：
```
方案A (2D嵌入) → 多角色对话剧 → 场景背景生成 → 2.5D 剧
```

不一步到位做真 3D 的原因：
1. 3D 渲染需要 GPU 资源远超现有系统
2. MLX 生态的 3D 支持有限
3. 用户对 2.5D 质量的接受度足够
4. 2D 数字人已有经过验证的流程

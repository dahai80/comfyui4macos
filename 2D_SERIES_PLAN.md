# AI 2D 30 分钟连续剧生成方案

> 固定角色、固定声音、数字人入剧、多集人物一致性
> 基于现有 comfyui4macos Pipeline 扩展

---

## 一、目标

在现有 Pipeline 基础上，实现 **30 分钟连续剧**的 AI 全自动生成，满足：

| 需求 | 说明 |
|:-----|:------|
| **定角** | 每部剧固定几个角色（主角/配角），不能每集换脸 |
| **定声** | 每个角色绑定固定声音（克隆声或预设声） |
| **入剧** | 用户创建的 2D 数字人形象可作为角色加入剧中 |
| **一致性** | 多集之间人物外貌、神态、声音保持一致 |
| **30分钟** | 全剧约 120-180 个场景，共 30 分钟 |

#### 角色绑定数据结构

```typescript
interface SeriesCharacter {
    name: string;           // 角色名（如"李明"）
    role: "protagonist" | "supporter";
    avatar_id: string | null;    // 绑定的数字人 ID
    voice_id: string | null;     // 绑定的声音 ID
    appearance: string;          // 外貌描述（自动生成或手动填）
    voice_description: string;   // 声音描述
    gender: "male" | "female";
}
```

### 4.2 数字人扮演角色流程

```
用户创建 2D 数字人（AvatarCreateStage）
         ↓
绑定到角色（角色配置面板）
         ↓
每场景生成时：
  1. PromptExpand 输出场景 JSON（含角色名）
  2. Pipeline 读取角色绑定 → 找到 avatar_id/voice_id
  3. 场景图像生成时 visual_prompt 注入角色外貌描述
  4. TTS 合成时使用角色绑定的声音
  5. AvatarAnimateStage 用绑定的数字人形象渲染
```

### 4.3 多角色同屏渲染（新增）

当前 AvatarAnimateStage 每场景只渲染一个角色。多角色对话场景需要：

```python
def render_multi_character_scene(
    scene: dict,
    character_bindings: dict,  # {角色名: {avatar_id, voice_id}}
    background: str,           # Flux 生成的场景图
) -> str:                      # 输出 clip 路径
    
    # 1. 场景中有几个角色说话，就渲染几个口型视频
    character_clips = []
    for char_name in scene["speaking_characters"]:
        binding = character_bindings[char_name]
        audio = tts_synthesize(scene["audio_script"], binding["voice_id"])
        clip = avatar_animate(
            ref_path=binding["avatar_ref"],
            audio_path=audio,
            duration=scene["duration"],
        )
        character_clips.append((char_name, clip, binding["position"]))
    
    # 2. 用 ffmpeg 多轨道合成到场景背景上
    final_clip = multi_layer_composite(
        background=background,
        layers=character_clips,  # 每个角色一个轨道
    )
    return final_clip
```

#### 同屏布局策略

| 角色数 | 布局 | 示意图 |
|:------:|:-----|:-------|
| 1人 | 居中 | `[    👤    ]` |
| 2人 | 左右 | `[ 👤 | 👤 ]` 或 `[ 👤 → → 👤 ]` |
| 3人 | 品字 | `[  👤  ] [👤|👤]` |

### 4.5 长视频渲染策略

```
30 分钟 ≈ 150 场景（平均 12s/场景）

渲染策略：
  1. 逐场景串行渲染（每场景 ~1-2 分钟）
  2. 每完成 10 场景保存一次 checkpoint
  3. 支持断点续跑
  4. 用户可随时查看已完成的片段

总渲染时间估算：
  ┌─ 场景背景 (Flux 8步)  30s/scene × 150 = 75min
  ├─ 角色图像 (Flux 8步)  30s/scene × 150 = 75min
  ├─ TTS 合成              5s/scene × 150 = 12.5min
  ├─ AvatarAnimate         20s/scene × 150 = 50min
  ├─ ffmpeg 合成            5s/scene × 150 = 12.5min
  └─ Assemble concat        10s
  ───────────────────────────────────────
  总计：约 225 分钟（3.75 小时）
```

## 五、分阶段实施

### Phase 1：角色绑定系统（1 周）

| 模块 | 内容 |
|:-----|:------|
| 前端 UI | 角色配置面板（添加/编辑/删除角色、绑定数字人、绑定声音） |
| 后端 API | character_registry 持久化到 checkpoint |
| Pipeline 增强 | PromptExpand 输出包含角色名，TTSSynthesizeStage 读取角色绑定 |
| 数据结构 | SeriesCharacter 类型定义、绑定存储 |

### Phase 2：数字人入剧（1 周）

| 模块 | 内容 |
|:-----|:------|
| AvatarAnimateStage 增强 | 支持从 ctx.artifacts 读取数字人参考图 |
| 场景图像生成 | 使用角色外貌描述注入 visual_prompt，实现换脸/换装 |
| 声音绑定 | TTS 自动使用角色绑定的声音克隆 |
| 验证 | 单角色剧集端到端测试 |

### Phase 3：多角色同屏（1 周）

| 模块 | 内容 |
|:-----|:------|
| 多角色渲染引擎 | 同一场景中渲染多个数字人 |
| ffmpeg 多轨道合成 | 画中画/分屏/左右布局 |
| 对话编排 | 根据 audio_script 自动切换说话角色 |
| 验证 | 双角色对话场景端到端测试 |

### Phase 4：30 分钟优化（1 周）

| 模块 | 内容 |
|:-----|:------|
| 分块渲染 | 每 10 场景 checkpoint，支持续跑 |
| 并行渲染 | ProcessPoolExecutor 并行渲染场景 |
| 快速预览 | 低分辨率快速生成预览版 |
| 前端进度 | 实时显示渲染进度和剩余时间 |

## 七、用户操作流程

```
1. 打开「创作新片」→ 选择「连续剧(Pro)」内容类型
2. 配置角色：
   a. 添加角色 → 输入名字
   b. 选择数字人形象（从「个人资产」选）
   c. 选择声音（从「个人资产」选）
   d. 设置角色外貌描述
3. 输入故事种子或上传 PDF
4. 点击「开始生成」
5. 切换到「作品集」页签查看进度
6. 每个场景完成后可预览
7. 全部完成 → 下载 30 分钟完整视频
```

| 风险 | 概率 | 影响 | 应对 |

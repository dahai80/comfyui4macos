# 数字人视频「AI 感」去除技术方案

> 让数字人看起来像真人拍摄，而不是 AI 生成的口播

---

## 一、为什么数字人视频一看就是 AI 生成的

### 1.1 当前视频的「AI 感」来源

```
┌────────────────────────────────────────────────────┐
│                 AI 感来源                          │
├────────────────────────────────────────────────────┤
│                                                    │
│  1. 只有嘴在动，脸完全静止 ← 最明显                 │
│     └─ 人眼对脸部异常极其敏感                       │
│                                                    │
│  2. 没有眼神变化 ← 第二明显                         │
│     └─ 正常人说话会眨眼、眼球转动                   │
│                                                    │
│  3. 没有头部自然晃动                                │
│     └─ 真人在说话时有 0.5-2° 的随机头部微动         │
│                                                    │
│  4. 背景是静态图片                                  │
│     └─ 即使有 Ken Burns，也是机械的                 │
│                                                    │
│  5. 口型过于"完美"                                  │
│     └─ 真人口型不会完全匹配每个音                   │
│                                                    │
│  6. 声音太干净 ← 已有去AI味功能                     │
│     └─ 加上混响和噪声后有改善                       │
│                                                    │
└────────────────────────────────────────────────────┘
```

### 1.2 对比真人视频

| 特征 | 真人拍摄 | 我们的数字人 | 差距 |
|:-----|:--------:|:-----------:|:----:|
| 头部微动 | ✅ 0.5-2°随机晃动 | ❌ 只有微动(±1px) | 极不自然 |
| 眨眼频率 | ✅ 每 3-5 秒一次 | ❌ 从不眨眼 | 僵尸感 |
| 眼神变化 | ✅ 看镜头、看稿、思考 | ❌ 一直盯镜头 | 不自然 |
| 眉毛活动 | ✅ 伴随语气变化 | ❌ 不动 | 表情僵硬 |
| 唇部细节 | ✅ 不完全匹配每个音 | ✅ 有口型但太干净 | 缺少杂音 |
| 呼吸感 | ✅ 胸部起伏、嘴唇微张 | ❌ 完全静止 | 不像活人 |
| 手部动作 | ✅ 自然手势 | ❌ 没有手 | 大头贴 |
| 音频 | ✅ 呼吸声、口水声 | ⚠️ 加了混响和噪声 | 有改善仍不够 |
| 背景 | ✅ 有深度、光影变化 | ❌ 静态背景 | 一眼假 |

## 二、短期可落地的优化（1-2 周）

### 2.1 面部微表情系统（最高优先级）

在 `_animate_mouth` 之外，增加完整的**面部微动系统**：

```python
def _apply_facial_micro_movements(
    frame: np.ndarray,
    t: float,
    energy: float,
    avatar_meta: dict,
) -> np.ndarray:
    """
    面部微表情引擎 — 让人脸看起来像活人
    """
    # 1. 眨眼（最重要！）
    # 每 3-5 秒随机眨眼一次
    # 眨眼持续 100-150ms（约 3 帧）
    blink_interval = 3.0 + random.random() * 2.0  # 3-5秒
    if t % blink_interval < 0.15:  # 眨眼中
        frame = _apply_blink(frame, avatar_meta["left_eye"], avatar_meta["right_eye"])
    
    # 2. 眼球微动（说话时眼球轻微转动）
    eye_movement = _compute_eye_movement(t, energy)
    frame = _apply_eye_shift(frame, avatar_meta["eyes"], eye_movement)
    
    # 3. 眉毛活动（伴随语气变化）
    brow_raise = energy * 0.3  # 高能量时眉毛上抬
    frame = _animate_eyebrows(frame, avatar_meta["eyebrows"], brow_raise)
    
    # 4. 头部微动（0.5-2° 随机晃动）
    head_movement = _compute_head_movement(t)
    frame = _apply_head_nod(frame, head_movement)
    
    # 5. 呼吸感（胸部区域轻微起伏）
    if has_chest_region(frame):
        frame = _apply_breathing(frame, t)
    
    return frame
```

### 2.2 各模块实现细节

#### 2.2.1 眨眼系统

```python
def _apply_blink(frame, left_eye_region, right_eye_region):
    """
    通过上眼睑下移模拟眨眼
    - 自然眨眼持续 100-150ms（24fps 下 3-4 帧）
    - 先慢合（2帧）→ 全闭（1帧）→ 慢开（2帧）
    """
    # 不修改眼睛图片，而是用肤色覆盖眼珠上半部分
    skin_color = _sample_eyelid_color(frame, left_eye_region)
    
    # 上眼睑下移覆盖 70% 的眼球
    frame = _draw_eyelid(frame, left_eye_region, skin_color, cover_ratio=0.7)
    frame = _draw_eyelid(frame, right_eye_region, skin_color, cover_ratio=0.7)
    
    return frame
```

#### 2.2.2 头部微动系统

```python
class HeadMovementController:
    """
    基于噪声函数（Perlin/Simplex）生成自然的头部运动
    - 使用 Perlin 噪声替代 sin/cos（太规律，一眼假）
    - 频率范围：0.5-3 Hz（慢速晃动）
    - 幅度范围：0.5-2.5 像素
    """
    def __init__(self, seed=42):
        self.noise = PerlinNoise(seed)
        self.t_offset = random.random() * 100  # 每个场景不同的相位
    
    def get_movement(self, t: float) -> tuple:
        """返回 (dx, dy, rotation)"""
        n1 = self.noise.noise(t * 0.5 + self.t_offset) * 1.5  # 水平
        n2 = self.noise.noise(t * 0.3 + self.t_offset + 50) * 1.0  # 垂直
        n3 = self.noise.noise(t * 0.4 + self.t_offset + 100) * 0.3  # 旋转
        return (n1, n2, n3)
```

## 六、技术难点与风险

| 难点 | 风险 | 应对 |
|:-----|:----:|:-----|
| 眨眼位置不准 | 眼睑覆盖到眼白外 | 用 avatar_meta 精确标定 |
| 头部微动过大 | 画面抖动 | 限制幅度 < 2.5px |
| Perlin 噪声计算 | CPU 开销 | 预计算 + 缓存 |
| 多模块协调 | 面部各部分动作不协调 | 统一时间轴控制 |
| 参考图质量 | 低质量图微表情效果差 | 提高人脸检测质量 |
